Observability & Monitoring

The Advanced Echo Server supports observability through Prometheus metrics, structured logging, and dedicated health check endpoints.

Prometheus Metrics

The server exposes metrics at the /metrics endpoint:

Custom Metrics

echo_requests_total

Counter of total HTTP requests

Labels: method, path, status
echo_request_duration_seconds

Histogram of request latency in seconds

No labels
echo_chaos_errors_total

Counter of chaos-induced errors

Labels: type

Default Go Metrics

Prometheus automatically exposes default Go runtime metrics such as go_goroutines, go_memstats_heap_alloc_bytes, process_cpu_seconds_total, and others.

Health Checks

Use these endpoints to check server status:

Liveness Check

# Simple liveness check
curl http://localhost:8080/health

Readiness Check

# Readiness probe for Kubernetes
curl http://localhost:8080/ready

Structured Logging

Logs are produced in a structured format with configurable levels:

Log Configuration

Environment Variable Description Default Options
LOG_LEVEL Minimum log level info debug, info, warn, error
LOG_FORMAT Log output format json json, text
LOG_REQUESTS Log all HTTP requests true true, false
LOG_BODY Include request/response bodies false true, false
